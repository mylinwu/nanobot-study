# 10｜术语表与常见问题

## 术语表

- **Agent**：可感知、可决策、可执行的智能体。
- **Planner**：任务规划器，把目标拆解为步骤。
- **Orchestrator**：流程调度器，管理状态与执行顺序。
- **Executor**：执行器，负责调用模型和工具。
- **RAG**：检索增强生成，用外部证据提升回答质量。
- **Reflection**：反思/评估模块，对结果做质量检查。
- **Guardrail**：护栏机制，保证安全、格式、合规。

## FAQ

### Q1：为什么不让一个大模型一次性做完？
因为复杂任务需要可控性、可审计性、可恢复性，单次生成通常无法满足。

### Q2：什么时候该引入多 Agent？
当任务出现“角色分工明显、步骤复杂、需要并行处理”时，多 Agent 更有价值。

### Q3：如何判断系统是否真的变好？
不要只看主观“像不像”，要看固定评测集上的准确率、完成率、时延和成本。

### Q4：学习这类项目最容易卡在哪？
通常卡在“只看代码，不画流程图”。建议先画控制流和时序图，再读代码。
