# 10｜术语表与常见问题

> 这一章是“查漏补缺”用的。建议在学习中反复回看。

## 1. 术语表（带通俗解释）

- **Agent**：能理解目标、做决策、调工具完成任务的智能体。
- **Planner**：把“大目标”拆成“小步骤”的模块。
- **Orchestrator**：负责派单、盯状态、做重试和恢复的调度中枢。
- **Executor**：真正干活的执行器，调用模型与工具产出结果。
- **RAG**：先检索证据再生成答案，减少“拍脑袋”输出。
- **Reflection**：对结果做质检（是否完整、正确、合规）。
- **Guardrail**：输出安全护栏（权限、格式、敏感内容约束）。
- **Idempotency（幂等）**：同一个请求重复执行，不应产生重复副作用。
- **Fallback（降级）**：主方案失败时切到备选方案确保可用性。

## 2. FAQ（从学习者视角）

### Q1：为什么不让一个大模型一次性做完？
因为复杂任务要求“可控、可追踪、可修复”。一次性生成通常不满足工程要求。

### Q2：什么时候该引入多 Agent？
当任务存在明显角色分工（如规划、检索、评估）且串行成本高时，再考虑多 Agent。

### Q3：我怎么判断自己真的学会了？
你至少要能独立完成三件事：

1. 画出主链路流程图；
2. 追踪一次失败任务并定位根因；
3. 提出一个可验证的优化方案并定义指标。

### Q4：最容易被忽略但最关键的能力是什么？
是“失败处理设计”。

很多系统平时看起来没问题，一到复杂场景就崩，根因几乎都在异常路径没设计好。

### Q5：教程里的 V1~V4 应该怎么实践？
推荐方式：

- 先在本地实现 V1，确保最小可运行；
- 每次只升级一个能力（如加入重试）；
- 每升一级都记录“指标变化”。

这比一次性追求“终极架构”更有效。
