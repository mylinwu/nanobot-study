# NanoBot 学习文档（中文教学版）

> 本教程针对“想真正看懂 Agent 项目实现”的学习者设计，不是概览大纲，而是**可落地的渐进式教程**。

## 学习目标

1. 搭建完整心智模型：用户请求如何变成可执行任务。
2. 理解关键模块存在的“工程原因”，而不是背名词。
3. 掌握模块从 V1 到 V4 的演进逻辑（每个模块至少三版）。
4. 能对照源码定位：入口、计划、执行、记忆、评估、运维。

---

## 如何使用这套教程（强烈建议）

- 不要按“看小说”的方式阅读，建议每章都输出一份自己的笔记。
- 每章至少做一次“伪代码复现”或“源码定位”。
- 遇到新概念时先看术语表，再回到章节上下文。
- 每学完 2 章，回到第 07 章的演进总表复盘一次。

## 教程验收标准（你可以对照自评）

完成这套教程后，你应该能：

1. 用 5~10 分钟讲清 Agent 主链路。
2. 独立说明任一模块从 V1 到 V4 的升级原因。
3. 追踪并解释一次失败任务的修复路径。
4. 为一个优化提案定义“质量/时延/成本”指标。
