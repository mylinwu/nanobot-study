# 05-2｜RAG：召回、重排与证据绑定

## 1. 最小链路

`query -> recall(topK) -> answer`

问题：相关不等于可回答。

## 2. 升级链路

`query rewrite -> hybrid recall -> rerank -> evidence pack -> answer`

## 3. 证据绑定

输出时要求：关键结论必须附 `evidence_id`。

```json
{"claim": "框架A部署成本更低", "evidence_id": ["doc12#p3", "doc18#p1"]}
```

## 4. 指标

- Recall@K
- nDCG
- 引用命中率
- 无证据结论率（越低越好）

## 5. 练习

给一个结论写出最少 2 条支持证据，并说明为何可信。
