# 08｜部署、监控、评测与成本优化

## 1. 部署建议

- 将控制面与执行面拆分部署。
- 队列化执行任务，避免高峰打爆服务。
- 使用配置中心管理模型、工具、密钥。

---

## 2. 监控指标（必须有）

- 任务成功率
- 平均时延 / P95 时延
- 每任务 token 成本
- 工具调用失败率
- 重试次数与原因分布

---

## 3. 评测体系

构建固定评测集，维度至少包括：

- 准确性（是否答对）
- 完整性（是否漏项）
- 可验证性（是否有证据）
- 稳定性（多次结果一致性）

---

## 4. 成本优化

- 模型分级：简单任务走便宜模型。
- 检索前置：先召回再生成，减少无效 token。
- 结果缓存：相同请求命中缓存。
- Prompt 压缩：去冗余上下文。
