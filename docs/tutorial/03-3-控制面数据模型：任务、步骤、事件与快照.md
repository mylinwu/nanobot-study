# 03-3｜控制面数据模型：任务、步骤、事件与快照

## 1. 为什么要先定义数据模型

控制面最大的问题不是“能不能跑”，而是“跑错后能不能还原现场”。

## 2. 四类核心对象

- Task：任务主对象（goal、owner、priority）。
- Step：步骤对象（status、attempt、depends_on）。
- Event：事件流（STEP_STARTED、STEP_FAILED、REPLAN_TRIGGERED）。
- Snapshot：快照（用于恢复）。

## 3. 最小字段建议

```json
{
  "task_id": "t_1001",
  "plan_version": 2,
  "step_id": "s3",
  "status": "RETRYING",
  "attempt": 2,
  "last_error": "tool_timeout",
  "updated_at": "2026-03-05T10:00:00Z"
}
```

## 4. 练习

给出一个“重规划前后 plan_version 变化”的事件序列（至少 6 条事件）。
